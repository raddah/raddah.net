كتيب تشغيل raddah.net

دليل شامل خطوة بخطوة لنشر المحتوى بالعربية والإنجليزية والصينية، ورفعه على GitHub، وربطه بـ Cloudflare عبر Astro.

١. مقدمة

هذا الموقع مدونة شخصية ثلاثية اللغات (عربي / إنجليزي / صيني) مبنية بإطار Astro وتُستضاف على Cloudflare Pages. الهدف من هذا الكتيب هو توفير مرجع واضح لكل عملية نشر — من كتابة المقال حتى ظهوره على الإنترنت.

ملاحظة: الموقع يستخدم نظام "Content Collections" في Astro، مما يعني أن كل مقالة هي ملف Markdown/MDX منفصل مع بيانات Frontmatter (عنوان، وصف، تاريخ، تصنيف... إلخ).

٢. هيكل المشروع

raddah-astro-dev/
├── src/
│   ├── content/blog/        ← مقالات المدونة
│   │   ├── ar/              ← النسخة العربية
│   │   ├── en/              ← النسخة الإنجليزية
│   │   └── zh/              ← النسخة الصينية
│   ├── pages/               ← صفحات Astro
│   │   ├── ar/ | en/ | zh/  ← كل لغة لها مجلد
│   ├── i18n/                ← الترجمة والإعدادات
│   ├── components/          ← المكونات
│   ├── layouts/             ← قوالب الصفحات
│   └── styles/              ← CSS المخصص
├── public/                  ← الصور والملفات الثابتة
├── scripts/new-post.ts      ← سكريبت إنشاء المقالات
├── astro.config.mjs         ← إعدادات Astro
└── .github/workflows/       ← GitHub Actions (النشر التلقائي)

٣. إنشاء مقالة جديدة

هناك طريقتان: الطريقة التلقائية (موصى بها) واليدوية.

الطريقة الأولى: السكريبت التلقائي

١
تشغيل الأمر
افتح الطرفية في مجلد المشروع ونفّذ:
npm run new:post
٢
اختيار اللغة
ستظهر قائمة بثلاث خيارات:
  • العربية (ar)
  • English (en)
  • 中文 (zh)
٣
إدخال البيانات
أدخل العنوان، الوصف القصير، التصنيف، والوسوم (مفصولة بفواصل). إذا كانت اللغة عربية، سيُطلب منك إدخال slug بالإنجليزية.
٤
التحقق من الإنشاء
سيتم إنشاء ملف Markdown في المسال الصحيح، مثال:
src/content/blog/ar/my-new-post.md

الطريقة الثانية: يدويًا

١
إنشاء الملف
أنشئ ملف .md داخل src/content/blog/XX/ حيث XX هي رمز اللغة.
---
title: "عنوان المقالة"
description: "وصف قصير"
publishDate: 2026-05-09
author: "Raddah"
category: "تقنية"
tags: ["astro", "مدونة"]
featured: false
draft: true
translations: {}
---

اكتب المحتوى هنا بصيغة Markdown...
تلميح: اترك draft: true أثناء الكتابة، وغيّره إلى false عندما تصبح جاهزًا للنشر.

٤. الترجمة للثلاث لغات

الموقع يدعم المحتوى المتوازي: كل مقالة يمكن أن تكون لها نسخ في اللغات الأخرى.

ربط الترجمات ببعضها

في الـ Frontmatter، استخدم حقل translations لربط النسخ المختلفة بنفس المقالة:

---
title: "عنوان المقالة العربي"
translations:
  en: "english-slug"
  zh: "chinese-slug"
---

الترجمة النصية للواجهة

كل النصوص الثابتة (القائمة، الأزرار، التذييل...) توجد في:

src/i18n/ui.ts

عند إضافة نص جديد، يجب إضافته للثلاث لغات:

'nav.newItem': 'عنصر جديد',      // ar
'nav.newItem': 'New Item',       // en
'nav.newItem': '新项目',           // zh
تنبيه: إذا أضفت نصًا جديدًا في ui.ts ولم تترجمه لإحدى اللغات، سيظهر بشكل غير صحيح في تلك النسخة.

٥. من المسودة إلى النشر

١
كتابة المحتوى
استخدم Markdown العادي أو MDX. يدعم الموقع: جداول، أكواد ملونة، اقتباسات، وصور.
٢
معاينة محلية
شغّل خادم التطوير:
npm run dev
ثم افتح http://localhost:4321
٣
التحقق من الأخطاء
نفّذ فحص الأنواع (TypeScript) قبل البناء:
npm run check
٤
بناء الموقع
تأكد أن البناء ينجح بدون أخطاء:
npm run build
٥
نشر المقالة
غيّر draft: true إلى draft: false في الملف.

٦. رفع التحديثات على GitHub

المشروع مرتبط بمستودع Git. كل تعديل يتم رفعه يُشغّل عملية بناء ونشر تلقائيًا.

الخطوات

١
مراجعة التغييرات
git status
٢
إضافة الملفات
git add src/content/blog/ar/my-post.md
أو أضف كل شيء:
git add .
٣
إنشاء Commit
git commit -m "إضافة مقالة: عنوان المقالة"
٤
الرفع إلى GitHub
git push origin main
نصيحة: اجعل رسالة Commit واضحة ووصفية، مثل: "add: agi-era post (ar/en/zh)".

٧. الربط مع Cloudflare

عند كل git push على فرع main، يعمل GitHub Action تلقائيًا لبناء الموقع ونشره على Cloudflare Pages.

كيف يعمل؟

  1. GitHub يستقبل التعديلات.
  2. يعمل ملف .github/workflows/deploy.yml.
  3. يثبت الاعتماديات: npm ci.
  4. يفحص الأنواع: npx astro check.
  5. يبني الموقع: npm run build.
  6. يرفع مجلد dist/ إلى Cloudflare Pages.

الإعدادات المطلوبة في GitHub

في إعدادات المستودع (Settings → Secrets and variables → Actions) يجب إضافة:

الاسم الوصف
CLOUDFLARE_API_TOKEN مفتاح API من Cloudflare (صلاحيات: Cloudflare Pages:Edit)
CLOUDFLARE_ACCOUNT_ID معرف حساب Cloudflare
تلميح: يمكنك مراقبة حالة البناء من تبويب Actions في مستودع GitHub. إذا فشل البناء، ستجد تفاصيل الخطأ هناك.

٨. المميزات المستخدمة في Astro

المحتوى

  • مُفعّل Content Collections مع تحقق Zod
  • مُفعّل i18n Routing (ar/en/zh)
  • مُفعّل MDX لدعم المكونات داخل المقالات
  • مُفعّل RSS Feed لكل لغة
  • مُفعّل Sitemap دولي

الأداء والتصميم

  • مُفعّل صور محسّنة عبر Sharp
  • مُفعّل Shiki + rehype-pretty-code (أكواد ملونة)
  • مُفعّل Pagefind (بحث فوري)
  • مُفعّل CSS مخصص مع متغيرات التصميم
  • مُفعّل IntersectionObserver لجدول المحتويات

الأتمتة

  • مُفعّل GitHub Actions للنشر التلقائي
  • مُفعّل سكريبت CLI لإنشاء المقالات
  • مُفعّل Prettier لتنسيق الكود
  • مُفعّل TypeScriptstrict

UX / واجهة المستخدم

  • مُفعّل تبديل الوضع الفاتح/الداكن
  • مُفعّل مؤشر تقدم القراءة
  • مُفعّل أزرار المشاركة (X / LinkedIn / نسخ)
  • مُفعّل مقالات ذات صلة
  • مُفعّل تخطيط RTL/LTR حسب اللغة

٩. مميزات Astro لم تُستخدم

هذه ميزات متاحة في Astro لكن المشروع الحالي لا يستخدمها — يمكنك إضافتها مستقبلًا إذا احتجتها:

الميزة السبب / البديل الحالي
Server Islands الموقع static بالكامل؛ لا حاجة لجلب ديناميكي من الخادم.
View Transitions API لم تُفعل؛ التنقل الحالي تقليدي سريع بما يكفي.
Astro DB لا توجد بيانات ديناميكية أو تعليقات أو مستخدمين.
Content Layer (الجديد) يُستخدم glob loader التقليدي؛ يمكن الترقية مستقبلًا.
React / Vue / Svelte كل المكونات مكتوبة بـ Astro纯 (zero JS) لأقصى سرعة.
Edge Middleware لا حاجة لتحويلات على الطلبات؛ static site كافٍ.
Partytown لا توجد سكريبتات طرف ثالث ثقيلة (Google Analytics إلخ).

١٠. حل المشكلات الشائعة

❌ البناء يفشل مع "Type error"

npm run check

اقرأ رسالة الخطأ وصحّح نوع البيانات في Frontmatter. غالبًا يكون سببه:

  • تاريخ غير صالح في publishDate
  • حقل مفقود (مثل category)

❌ المقالة لا تظهر في الموقع

  • تأكد أن draft: false
  • تأكد أن الملف في المجلد الصحيح: src/content/blog/XX/
  • تأكد من صحة الامتداد .md أو .mdx

❌ الترجمة لا تظهر في Language Switcher

  • تأكد من ربط translations في Frontmatter.
  • تأكد أن slug اللغة الأخرى مطابق لاسم الملف.

❌ Pagefind (البحث) لا يعمل محليًا

البحث يحتاج بناءً أولًا لإنشاء الفهرس:
npm run build
ثم:
npm run preview

❌ GitHub Action فشل في النشر

  • تحقق من صحة CLOUDFLARE_API_TOKEN وصلاحياته.
  • تأكد أن CLOUDFLARE_ACCOUNT_ID صحيح.
  • راجع سجلّ Actions في GitHub للتفاصيل.

١١. قائمة التحقق السريعة (Checklist)

قبل كل نشر، راجع هذه القائمة:

  • ☐ المقالة مكتوبة بصيغة Markdown/MDX صحيحة
  • ☐ Frontmatter كامل (title, description, publishDate, category)
  • ☐ draft: false للمقالات الجاهزة
  • ☐ الترجمة موجودة لكل اللغات المطلوبة (إن وُجدت)
  • ☐ الصور موجودة في مجلد public/images/blog/
  • ☐ npm run check — لا أخطاء
  • ☐ npm run build — نجاح
  • ☐ git commit واضح ووصفي
  • ☐ git push origin main
  • ☐ التحقق من تبويب Actions في GitHub
  • ☐ فتح raddah.net والتأكد من ظهور المحتوى
بعد النشر: تحقق من العنوان على وسائل التواصل (Open Graph) باستخدام أداة مثل opengraph.xyz.
